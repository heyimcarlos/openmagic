# Use closed declarative Workflow Definitions

OpenMagic uses immutable, digest-verified Workflow Definitions instead of static-only DAGs, free reducers, or replayed orchestration code. Each Instance is permanently pinned to a separate Definition Key and integer Definition Version; typed Commands, canonical Step outcomes, and exact Wait-targeted Signals may activate only predefined named Routes that atomically materialize finite Step or Wait occurrences. This supports repeated human-authorized revisions and recovery from explicit durable state while keeping authorization and route selection in Business Policy and preventing callers, Agents, Workers, and Executors from inventing runtime graphs.

Steps are executable occurrences with opaque identity and exact AND-success dependencies. Waits are distinct, non-executable, one-shot gates with no Attempt, lease, retry, timer, or business output; one accepted Signal targets and satisfies one exact Wait, and time-based progression arrives through policy-authorized Trigger Signals. Route input binding is structural and typed, Definition registration fails closed, replays return the same occurrences, and active Instances never fall forward to another Definition version.
