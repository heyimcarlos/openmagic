Reading the OpenMagic eval prototype

The screen is a catalog of proposed evidence, not a test execution dashboard.

Read one scenario from top to bottom

Name: the behavior under examination.

Systems: which architecture is exercised.

Verdict: whether failure blocks delivery or only informs us.

Perturbation: the pressure applied to the system.

Evidence: the concrete observations required afterward.

The three verdicts

MarkerQuestion answeredExample
[g]Must V0 behave this way?No duplicate send after uncertain dispatch.
[d]What did we measure?Workflow mode loaded fewer context tokens.
[s]Does the real integration work now?One exact Gmail message reached AgentMail.

Navigate with a purpose

  1. Press l until the lane is paired journey.
  2. Press n through its four scenarios. Ask whether each comparison is fair.
  3. Press l for protocol recovery. Ask whether every safety invariant is represented.
  4. Press l for live provider. Notice that it proves only the normal happy path.

Retrieval exercise

Find Stop after uncertain dispatch. Before reading its verdict, decide whether it should be a gate, diagnostic, or smoke proof. Then explain why in one sentence.
